QA Platform Implementation Files

A comprehensive quality assurance platform built with Google Apps Script

infoInstructions

This page contains all the files needed to implement the QA Platform in Google Apps Script. Follow these steps to set up your platform:

  1. Open your Google Sheet that will serve as the backend for the QA Platform
  2. Go to Extensions > Apps Script to open the script editor
  3. For each file listed below, create a new script file with the same name and copy-paste the contents
  4. After adding all files, save your project
  5. Deploy the script by clicking "Deploy" > "New deployment"
  6. Choose "Web app" as the deployment type and configure access permissions
  7. Return to your Google Sheet and refresh the page
  8. Use the custom menu that appears to initialize the platform and begin using it

Note: Make sure you have the necessary permissions to create and run Apps Script in your Google Workspace environment.

warningImportant Deployment Note

When deploying as a web app, be sure to:

  • Set "Execute as" to either "User accessing the web app" or your account
  • Set "Who has access" according to your organization's requirements
  • The doGet() function has been included in Code.gs to serve this index page

codeCore Script Files

  • javascript Code.gs Main entry point. Contains menu creation and global constants.
  • database DatabaseOperations.gs Core database operations for interacting with the Google Sheet.
  • mail EmailService.gs Email notification functions for evaluations and disputes.
  • people UserManagement.gs User management and authentication functions.
  • assignment Evaluation.gs Evaluation creation and management functions.
  • gavel Dispute.gs Dispute creation and resolution functions.
  • dashboard Dashboard.gs Dashboard data and visualization functions.
  • settings Admin.gs Administration functions for settings and system management.
  • import_export ImportExport.gs Import/export functions for data exchange.

webHTML UI Files

  • menu UI/MainSidebar.html Main sidebar interface with navigation.
  • dashboard UI/Dashboard.html Dashboard interface with metrics and charts.
  • rate_review UI/EvaluationForm.html Evaluation form for QA analysts.
  • gavel UI/DisputeForm.html Dispute form for managers.
  • verified UI/DisputeReview.html Dispute review interface for QA managers.
  • settings UI/AdminPanel.html Admin panel for system configuration.
  • people UI/UserManagement.html User management interface.
  • style UI/styles.html CSS styles for the UI components.
  • code UI/scripts.html Common JavaScript functions for the UI.
